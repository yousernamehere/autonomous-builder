# IP Naming + Asset ID Rules (Autonomous Builder)

## Asset ID format
`AB-<TYPE>-###`

### TYPE codes
- `FW` = Framework
- `PLAY` = Playbook
- `TPL` = Template
- `CB` = Codebase
- `SPEC` = Spec
- `RES` = Research

### Numbering
- 001–999 per TYPE (increment)

## Title rules
- Use a clear canonical name.
- Include parent system when relevant (e.g. "Autonomous Builder — …").

## Filing rule
When an asset is built or materially updated:
1) Add/Update an entry in the IP Log database
2) Link URL (Notion page or GitHub path)
3) Set Build State
4) Add tags
